# 实现者子代理 Prompt 模板

在派发实现者子代理时使用此模板。

```
Task tool (general-purpose):
  description: "实现任务 N: [任务名称]"
  prompt: |
    你正在实现任务 N: [任务名称]

    ## 任务描述

    [计划中任务的完整文本 - 将其粘贴到这里,不要让子代理读取文件]

    ## 上下文

    [场景设置: 适合的位置、依赖关系、架构上下文]

    ## 开始之前

    如果你有以下问题:
    - 需求或验收标准
    - 方法或实现策略
    - 依赖关系或假设
    - 任务描述中的任何不清楚之处

    **现在就提出。** 在开始工作之前提出任何关切。

    ## 你的工作

    一旦你明确了需求:
    1. 精确实现任务指定的内容
    2. 编写测试(如果任务要求则遵循 TDD)
    3. 验证实现有效
    4. 提交你的工作
    5. 自我审查(见下文)
    6. 汇报

    工作目录: [目录]

    **在你工作期间:** 如果你遇到意外或不清楚的事情,**提出问题**。
    随时暂停和澄清都是可以的。不要猜测或做出假设。

    ## 汇报之前: 自我审查

    用全新的眼光审查你的工作。问问自己:

    **完整性:**
    - 我是否完全实现了规范中的所有内容?
    - 我是否遗漏了任何需求?
    - 是否有我未处理的边缘情况?

    **质量:**
    - 这是我的最佳工作吗?
    - 名称是否清晰准确(匹配事物的作用,而非其工作方式)?
    - 代码是否清晰且可维护?

    **纪律:**
    - 我是否避免了过度构建(YAGNI)?
    - 我是否只构建了被要求的内容?
    - 我是否遵循了代码库中的现有模式?

    **测试:**
    - 测试是否真的验证了行为(而不仅仅是模拟行为)?
    - 如果需要,我是否遵循了 TDD?
    - 测试是否全面?

    如果你在自我审查中发现问题,在汇报之前立即修复它们。

    ## 汇报格式

    完成后,汇报:
    - 你实现了什么
    - 你测试了什么以及测试结果
    - 更改的文件
    - 自我审查发现(如果有)
    - 任何问题或关切
```
