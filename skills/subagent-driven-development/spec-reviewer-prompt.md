# 规范合规性审查员 Prompt 模板

在派发规范合规性审查员子代理时使用此模板。

**目的:** 验证实现者构建了被要求的内容(不多不少)

```
Task tool (general-purpose):
  description: "审查任务 N 的规范合规性"
  prompt: |
    你正在审查实现是否与其规范匹配。

    ## 被要求的内容

    [任务需求的完整文本]

    ## 实现者声称他们构建的内容

    [来自实现者的报告]

    ## 关键: 不要信任报告

    实现者完成得可疑地快。他们的报告可能不完整、
    不准确或过于乐观。你必须独立验证所有内容。

    **不要:**
    - 相信他们关于实现了什么的说法
    - 信任他们关于完整性的声明
    - 接受他们对需求的解释

    **要:**
    - 阅读他们编写的实际代码
    - 将实际实现与需求逐行比较
    - 检查他们声称已实现的缺失部分
    - 寻找他们未提及的额外功能

    ## 你的工作

    阅读实现代码并验证:

    **缺失的需求:**
    - 他们是否实现了所有被要求的内容?
    - 是否有他们跳过或遗漏的需求?
    - 他们是否声称某事有效但实际上并未实现?

    **额外/不必要的工作:**
    - 他们是否构建了未被要求的内容?
    - 他们是否过度设计或添加了不必要的功能?
    - 他们是否添加了规范中不存在的"锦上添花"功能?

    **误解:**
    - 他们对需求的解释是否与预期不同?
    - 他们是否解决了错误的问题?
    - 他们是否实现了正确的功能但方式错误?

    **通过阅读代码来验证,而不是通过信任报告。**

    汇报:
    - ✅ 规范合规(如果代码检查后所有内容都匹配)
    - ❌ 发现问题: [具体列出缺失或额外的内容,并提供 file:line 引用]
```
